Contribution guidelines

Toolipedia follows an editor-curated model inspired by Wikipedia's article review process. Community submissions are welcome but every tool goes through review before it's published.

What gets approved

  • Tools that are publicly available — not internal-only software
  • Tools with a clear AI component, not just software that uses AI incidentally
  • Tools with a working website and verifiable pricing
  • Tools not already in the directory under a different name
  • Descriptions written factually, not as marketing copy

What gets rejected or sent back

  • Tools in private beta with no public access
  • Duplicate submissions for tools already listed
  • Descriptions that read like press releases or ads
  • Missing required fields or unverifiable pricing

Content Policy & Ranking Policy

Toolipedia is an independent, editorially curated directory of AI tools. This page explains what gets listed, how tools are ranked, and what vendors can and cannot do.

Independence guarantee

Toolipedia operates with no commercial relationships with any of the tools it lists. No vendor pays to be included. No vendor pays for a higher position. No vendor pays for a better description. The directory earns nothing from listings — it exists to help users, not to generate revenue from vendors.

No pay-to-play. Listings are ordered by relevance and usefulness, not payment. Paying for a position is not possible — there is no mechanism to do it.

What qualifies for inclusion

A tool is considered for inclusion if it meets all of the following:

  • It is publicly accessible — not invite-only, internal-only, or limited to one organisation
  • It has a genuine AI component central to its value, not a minor feature added onto non-AI software
  • It has a live, working product with verifiable pricing
  • It is not already listed under a different name or URL
  • Its description can be written neutrally and factually

How tools are ranked

When a category filter is active, tools are ranked by their capability score for that category — a numeric rating across six dimensions (writing, coding, research, creativity, automation, multimodal). These scores are set by the editor based on publicly available benchmarks, user research, and direct testing. They are not influenced by vendors.

When no filter is active, tools appear in the order they were added. Within the AI Advisor chat, tools are recommended by matching relevance to the user's stated goal.

Ranking factors: category relevance score, breadth of genuine use cases, quality and reliability of the free tier (if any), strength of integrations, and community feedback. Payment is not a factor at any stage.

What vendors can do

  • Submit a tool for consideration using the Submit tab — treated like any other submission
  • Contact the editor to correct factual inaccuracies in an existing listing (pricing change, feature update, URL change)
  • Provide official documentation or release notes to support a correction request

What vendors cannot do

  • Pay for inclusion, a higher ranking, or a featured position
  • Write or dictate the description of their own tool
  • Request removal of a competitor's listing
  • Request that scores be adjusted without providing verifiable evidence
Corrections, not marketing. If a vendor contacts the editor, the conversation is limited to factual corrections — pricing, features, URLs. Vendors cannot write or approve the editorial copy that describes their tool.

Community submissions

Any user can submit a tool through the Submit tab. Submissions go into the review queue and are assessed against the inclusion criteria above. The editor makes the final decision. Submitting a tool does not guarantee inclusion.

If a submitted tool is approved, it joins the directory with a description written or edited by the editor — not copied directly from the submission or from the vendor's marketing materials.

Accuracy and updates

AI tool pricing and features change frequently. Toolipedia aims to review and update listings monthly. If you notice something that is out of date, use the Submit tab to flag it — select the existing tool's name and describe the correction needed.

Contact

For policy questions, factual corrections, or editorial feedback: submit through the community queue or reach the editor via GitHub at dishts04.
